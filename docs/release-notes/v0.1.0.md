# MarkEdit RepoTray v0.1.0

Date: 2026-03-09

## Highlights

- Adds the first RepoTray command for MarkEdit: `Open Linked Markdown`
- Parses Markdown links and wiki links from the current document
- Resolves linked notes relative to the current file
- Shows linked notes in a mounted quick-switcher overlay with active targets first, collapsed missing results, and compact status dots

## User Notes

- Missing targets are intentionally non-destructive in v0.1.0
- The picker works from a snapshot taken when the command opens
- Keyboard shortcut default is `Shift+Command+L`
- Filtering is available directly in the picker input
- Missing results stay collapsed behind `Missing (N)` until expanded or matched by the filter
- Existing targets must be in a folder MarkEdit can access; testing from `~/Documents` is more reliable than `/tmp`

## Packaging Notes

- Build artifact: `dist/markedit-repotray.js`
- Install target: `~/Library/Containers/app.cyan.markedit/Data/Documents/scripts/`
- Restart MarkEdit after copying the script into place

## Known Gaps

- Screenshot is still pending
- No file-creation flow for missing links in this release
- Live smoke-testing coverage is still limited beyond the verified accessible-folder path
